请求方式总结

发表于2025-10-20|更新于2025-10-25|小知识点
|浏览量:

非常棒 👍，这个问题很关键！要彻底弄懂前后端交互，必须清楚：

  1. 前端有哪些请求方式（method）
  2. 每种请求的参数怎么传递（path、query、body、header）
  3. 在前端（如 axios、fetch、Form）和后端（Spring）如何接收

下面我帮你系统总结成一份「前端请求方式与参数传递大全」👇

🚀 一、HTTP 常见请求方式（method）

请求方式 主要用途 是否有请求体（body） 是否幂等 典型场景
GET 获取数据 ❌ 无 ✅ 是 获取列表、详情
POST 创建数据 ✅ 有 ❌ 否 新增、登录、上传
PUT 更新整体数据 ✅ 有 ✅ 是 修改整条记录
PATCH 更新部分数据 ✅ 有 ✅ 是 局部更新
DELETE 删除数据 ✅ 可有可无 ✅ 是 删除资源

🧩 二、参数传递的四种方式（非常重要）

参数类型 位置 示例 前端写法 后端接收注解
Path 参数 URL 路径中 /users/1 /users/${id} @PathVariable
Query 参数 ? 后面 /users?page=1&size=10 { params: { page:1, size:10 }} @RequestParam
Body 参数 请求体中（JSON、Form） { "username": "Tom" } data: {...} @RequestBody
Header 参数 HTTP 头部 Authorization: Bearer xxx headers: { token: '...' } @RequestHeader
